## Bugs encontrados

### 🔴 BUG-01 (Crítico) — El Asistente IA está completamente roto (siempre 400)
**Dónde:** `src/components/Asistente/index.tsx` ↔ `src/app/api/chat/route.ts`
**Síntoma:** cualquier mensaje (chip de sugerencia o texto manual) responde `POST /api/chat → 400 "Mensaje requerido"`. No se renderiza ninguna respuesta ni el mensaje del usuario.
**Causa raíz:** desajuste de contrato del AI SDK.
- El frontend usa `useChat` + `DefaultChatTransport` (AI SDK v6), que envía el body como `{ messages: UIMessage[], id }`.
- El endpoint lee `const { id, message } = body` esperando un **string** `message`. Como `message` es `undefined` → corta con 400 en la línea 27.
- **Segundo desajuste latente:** el route devuelve `result.toTextStreamResponse()` (stream de texto plano), pero `useChat`/`DefaultChatTransport` espera el protocolo **UI Message Stream** (`toUIMessageStreamResponse()`). Aunque se arregle el body, los tool-calls no se renderizarían.
**Impacto:** la feature estrella ("Fase A" del último commit) no funciona end-to-end.
**Reproducción:** Asistente → clic en "Mostrame mis rutinas" → no pasa nada → Network: 400.

### 🟠 BUG-02 (Alto) — Las mutaciones no refrescan la UI (estado obsoleto sistémico)
**Dónde:** flujo de `useRoutines` / vistas de Rutinas y Editor.
**Síntoma 1:** "Usar esta plantilla" instala la rutina (el % de la home cambia, y tras **recargar** aparece en la lista), pero la lista de Rutinas **no se actualiza** sin recarga manual.
**Síntoma 2:** En el editor, "Agregar tarea" persiste en BD (tras recargar muestra `Tareas (6)`), pero el contador y la lista siguen en `Tareas (5)` hasta recargar.
**Impacto:** el usuario cree que la acción falló y la repite → riesgo de duplicados.
**Patrón:** falta invalidación/refetch o actualización optimista tras las mutaciones.

### 🟡 BUG-03 (Medio) — Mensajes de error de Supabase sin traducir
**Dónde:** `src/app/login/page.tsx`, `src/hooks/useAuth.tsx`.
**Síntoma:** en una UI 100% en español aparecen errores crudos en inglés:
- Login con credenciales erróneas → "Invalid login credentials".
- Registro con dominio rechazado → "Email address … is invalid".
- Registro con rate limit → "email rate limit exceeded".
**Nota:** `register` sí traduce el caso de email duplicado, pero ningún otro error está mapeado. Falta un mapeo central de errores de Supabase → español.

### 🟡 BUG-04 (Medio) — Validación de email inalcanzable (mensaje en inglés del navegador)
**Dónde:** `src/app/login/page.tsx` y `register/page.tsx`.
**Síntoma:** con email mal formado ("notanemail"), la validación nativa de `<input type="email" required>` intercepta el submit y muestra el tooltip del navegador en inglés ("Please include an '@'…"). La validación JS personalizada en español (`"Correo electrónico no válido"`) **nunca se ejecuta** (código muerto).
**Fix sugerido:** añadir `noValidate` al `<form>` para que corra la validación propia (como ya hace `RoutineEditor`), o eliminar la validación JS redundante.

### 🟢 BUG-05 (Bajo) — Inconsistencia de tono: voseo vs. tuteo
**Dónde:** Asistente (`Welcome.tsx`, system prompt) usa voseo rioplatense ("Mostrame", "Creá", "Agregá", "Escribí"), mientras el resto de la app usa tuteo neutro ("Prueba con otro", "Selecciona al menos un día", "Intenta de nuevo").
**Impacto:** identidad de marca inconsistente. Definir un único registro.

### 🟢 BUG-06 (Bajo / cosmético) — Fila de tema redundante en Ajustes
**Dónde:** `src/components/SettingsPanel.tsx:101-104`.
**Síntoma:** la fila muestra el label fijo "Modo oscuro" y, al lado, el valor `{isDark ? "Modo oscuro" : "Modo claro"}`. En modo claro se lee "Modo oscuro … Modo claro" (confuso). El label debería ser neutro ("Tema" / "Apariencia") y el valor reflejar el estado actual.

### 🔵 OBS-07 (A verificar) — Doble disparo de peticiones REST en dev
**Síntoma:** cada query a Supabase (`routines`, `tasks`, `task_completions`) se dispara **dos veces**.
**Probable causa:** React StrictMode (doble invocación en desarrollo) — normalmente inofensivo. **Verificar** que no se duplique en producción ni que cause doble escritura en mutaciones.

# Resolución — fixes aplicados (2026-06-21)

Todos los bugs se resolvieron y se verificaron en la app en ejecución. `tsc --noEmit` limpio y **283 tests pasan**.

### ✅ BUG-01 — Asistente IA (RESUELTO, tenía 5 causas en capas)
El asistente estaba roto por cinco motivos encadenados; al destapar uno aparecía el siguiente:

1. **Contrato del AI SDK.** El endpoint leía `message` (string); el cliente envía `messages: UIMessage[]`. → Reescrito [route.ts](src/app/api/chat/route.ts) para leer `messages`, usar `await convertToModelMessages(messages)` y responder con `toUIMessageStreamResponse({ originalMessages, onFinish, onError })`.
2. **Transport inestable.** `useChat` recibía un `new DefaultChatTransport()` y un `id` nuevos en cada render → recreaba el chat y **borraba los mensajes** (volvía a "Welcome"). → Memoizados `chatId` y `transport` con `useMemo` en [Asistente/index.tsx](src/components/Asistente/index.tsx).
3. **Modelo inexistente.** `gemini-2.0-flash` devuelve 404 ("no longer available"). → Actualizado a `gemini-2.5-flash`.
4. **Esquema de tools.** `createTool` construía `parameters`; el AI SDK v6 espera **`inputSchema`** → las tools nunca recibían sus argumentos y fallaban. → Corregido en [tools.ts](src/lib/agent/tools.ts).
5. **Sin contexto de grupo.** El system prompt prometía "group context" pero no lo inyectaba → las tools no sabían el `groupId`. → El endpoint resuelve el grupo activo (`ensurePersonalGroup` + `getActiveGroupId`) y lo inyecta en el prompt.

**Verificado en vivo:** el asistente renderiza los mensajes, llama a tools de lectura y escritura, y respondió en español; creó la rutina "Prueba IA" de extremo a extremo. Se añadió `onError` para mostrar un mensaje en español en vez de "An error occurred."

> ⚠️ **Pendiente de infraestructura (no es código):** la tabla `public.chat_sessions` no existe en la BD remota (`PGRST205`), por lo que la persistencia del historial falla de forma silenciosa (solo se loguea; no rompe la respuesta). Falta aplicar la migración [`supabase/migrations/20260621020000_chat_sessions.sql`](supabase/migrations/20260621020000_chat_sessions.sql) al proyecto remoto (`supabase db push`).

### ✅ BUG-02 — Refresco de mutaciones (RESUELTO)
`useRoutines` ahora expone `refetch`; `RoutineBuilder.refreshHome()` hace `await refetch()` + `router.refresh()`, de modo que la lista cliente se actualiza tras instalar plantillas, crear/editar/borrar tareas y reordenar. Eliminada la sincronización manual con estado obsoleto en `handleAddTask`.

### ✅ BUG-03 — Errores de Supabase traducidos (RESUELTO)
Nueva util [translateAuthError](src/lib/auth/authErrors.ts) usada por `useAuth` (login y signup). El registro ya no duplica el mapeo. Cubre credenciales inválidas, email inválido, rate limit, duplicado, longitud, email no confirmado, etc.

### ✅ BUG-04 — Validación de email en español (RESUELTO)
`noValidate` añadido a los forms de login y registro → corre la validación JS en español. **Verificado en vivo:** email mal formado ahora muestra "Correo electrónico no válido". Añadida validación cliente de longitud mínima (≥6) en el registro (antes la daba el `minLength` nativo, ahora desactivado).

### ✅ BUG-05 — Tono unificado a tuteo (RESUELTO)
Chips y placeholder del asistente y el system prompt pasados a tuteo neutro ("Muéstrame", "Crea", "Escribe", "Puedes", "Responde").

### ✅ BUG-06 — Fila de tema (RESUELTO)
